# 📘 Лабораторная работа №1

## Вариант 15

### Тема: Реализация gRPC-сервиса SurveyService

---

# 📌 Задание

Необходимо реализовать gRPC-сервис **SurveyService**, который позволяет получать список вопросов опроса и отправлять ответы пользователя.

Сервис должен содержать методы:


GetSurvey


Метод возвращает список вопросов опроса.


SubmitAnswer


Метод позволяет отправить ответ пользователя на вопрос.

---

# 🏗 Архитектура решения

В работе используется **клиент-серверная архитектура**.

Клиент отправляет запросы серверу через gRPC, а сервер обрабатывает их и возвращает ответы.


+-------------+ gRPC +-------------+
| Клиент | <---------------> | Сервер |
| client.py | | server.py |
+-------------+ +-------------+
|
|
+---------------+
| survey_data |
| (в памяти) |
+---------------+


## Компоненты системы

**client.py**

Клиентское приложение, которое подключается к серверу и выполняет вызовы методов сервиса.

**server.py**

Серверная часть приложения, реализующая обработку запросов и бизнес-логику.

**survey.proto**

Файл описания сервиса и структур данных с использованием **Protocol Buffers**.

**survey_data**

Имитация базы данных, содержащая список вопросов и ответов.

---

# 🧩 Описание реализации

Для описания сервиса используется файл:


survey.proto


В этом файле определяется сервис **SurveyService** и структуры сообщений.

Сервис содержит два метода:

- `GetSurvey` — получение списка вопросов опроса;
- `SubmitAnswer` — отправка ответа пользователя.

Метод `GetSurvey` возвращает список вопросов, которые клиент может отобразить пользователю.

Метод `SubmitAnswer` принимает:

- идентификатор вопроса
- текст ответа пользователя

После чего сервер сохраняет ответ.

# ⚙ Генерация gRPC-кода

После создания файла `survey.proto` была выполнена команда:


python -m grpc_tools.protoc -I. --python_out=. --grpc_python_out=. survey.proto

В результате были автоматически созданы файлы:

survey_pb2.py

survey_pb2_grpc.py

Эти файлы обеспечивают взаимодействие клиента и сервера через gRPC.

🖥 Реализация сервера

Серверная часть реализована в файле:

server.py

Создан класс:

SurveyService

который наследуется от:

SurveyServiceServicer

В этом классе реализована логика методов сервиса.

Метод GetSurvey

Метод возвращает список вопросов опроса.

Вопросы хранятся в виде списка строк в памяти программы.

При получении запроса сервер формирует ответ и отправляет клиенту список вопросов.

Метод SubmitAnswer

Метод принимает:

идентификатор вопроса

ответ пользователя

После получения данных сервер сохраняет ответ в памяти.

Сервер запускается на порту:

50051

и ожидает подключения клиентов.

💻 Реализация клиента

Клиентская часть реализована в файле:

client.py

Подключение к серверу выполняется через:

grpc.insecure_channel('localhost:50051')

Клиент выполняет следующие действия:

Подключается к серверу.

Получает список вопросов опроса.

Отправляет ответы пользователя.

После отправки ответа клиент получает подтверждение от сервера.

🚀 Запуск проекта

Для запуска проекта необходимо выполнить следующие шаги.

1. Активация виртуального окружения

python client.py

Клиент подключается к серверу и выполняет вызовы методов сервиса.
