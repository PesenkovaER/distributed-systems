# 🔬 Методология

В работе использовалась симуляция Gossip-протокола для распределенной системы, состоящей из нескольких географических кластеров.

Система включает:
- 200 узлов;
- 4 кластера по 50 узлов;
- обмен gossip-сообщениями между узлами;
- случайные потери пакетов;
- моделирование отказов узлов;
- межкластерные задержки передачи сообщений.

Исследовалось влияние следующих параметров:
- Gossip Interval — частота обмена сообщениями;
- Gossip Fanout — количество соседей для gossip-рассылки;
- Packet Loss — процент потерянных сообщений;
- межкластерная задержка.

Для моделирования использовался класс `GeoSerfSimulator`, расширяющий базовую модель Gossip-протокола с учетом межкластерных задержек.

## Анализ графика

На графике показано влияние Gossip Interval и Fanout на время полной конвергенции системы.

### Наблюдения

- При увеличении Gossip Interval время конвергенции увеличивается.
- Fanout = 5 обеспечивает более быстрое распространение информации.
- Минимальное время convergence достигается при Interval = 0.2 с.

При меньшем Gossip Interval узлы чаще обмениваются сообщениями, поэтому информация об отказе распространяется быстрее.

Больший Fanout позволяет каждому узлу отправлять gossip большему числу соседей, что также ускоряет распространение информации.

- Линии для Fanout=5 находятся выше линий для Fanout=3
- Разрыв между Loss=5% и Loss=10% постоянен для всех Interval


### Основные выводы по Bandwidth

1. **Gossip Interval** — наиболее влиятельный параметр: изменение с 0.2 с до 1.0 с снижает bandwidth в 5 раз

2. **Gossip Fanout** — увеличение с 3 до 5 повышает bandwidth в 1.67 раза

3. **Packet Loss** — потеря 10% пакетов снижает bandwidth на 10% относительно потери 5%

4. **Компромисс (Trade-off):**
   - **Высокий bandwidth** → быстрое обнаружение отказов, но высокая нагрузка на сеть
   - **Низкий bandwidth** → экономия трафика, но медленное распространение информации

5. **Рекомендация:** Для географически распределенных систем (вариант 15) с межкластерными задержками 50–200 мс рекомендуется выбирать конфигурацию с **умеренным bandwidth**, обеспечивающим баланс между скоростью конвергенции и нагрузкой на сеть.


## Какие параметры лучше?

Для варианта 15 (географические кластеры с задержками 50-200 мс):

| Цель | Interval | Fanout | Loss | Bandwidth |
|------|----------|--------|------|-----------|
| 🚀 **Максимальная скорость** | 0.2 с | 5 | 5% | 22 Мбит/с |
| ⚖️ **Баланс (рекомендую)** | 0.5 с | 5 | 5% | 8.8 Мбит/с |
| 💰 **Экономия трафика** | 1.0 с | 3 | 10% | 2.4 Мбит/с |

## Золотое правило

**Меньше Interval → Быстрее конвергенция → Больше Bandwidth**

**Больше Fanout → Быстрее конвергенция → Больше Bandwidth**

## Итог

Для географически распределенных систем лучше брать **маленький Interval (0.2-0.5 с)** и **большой Fanout (5)**, чтобы компенсировать межкластерные задержки. Рекомендуемая конфигурация: **Interval=0.5с, Fanout=5, Loss=5%** (Bandwidth = 8.8 Мбит/с).

# 📝 Выводы по работе

В ходе лабораторной работы был исследован Gossip-протокол обнаружения отказов в распределенных системах на примере географически распределенной сети из 200 узлов, разделенных на 4 кластера.

В процессе моделирования анализировалось влияние следующих параметров:
- Gossip Interval;
- Gossip Fanout;
- потери пакетов;
- межкластерные задержки.

Результаты симуляции показали, что уменьшение Gossip Interval значительно ускоряет распространение информации между узлами, однако увеличивает нагрузку на сеть. Увеличение Gossip Fanout также ускоряет convergence системы за счет рассылки сообщений большему числу соседей, но приводит к росту количества сетевых сообщений.

Было установлено, что Gossip-протокол сохраняет устойчивость даже при потерях пакетов 5–10%, поскольку информация распространяется избыточно и многократно дублируется между узлами.

Дополнительно было исследовано влияние географической распределенности системы. Увеличение межкластерных задержек существенно замедляет распространение информации между регионами и увеличивает время конвергенции всей системы.

В рамках дополнительной оптимизации параметров для задержек 50–200 мс была найдена наиболее эффективная конфигурация:
- межкластерная задержка — 50 мс;
- Gossip Interval — 0.2 с;
- Gossip Fanout — 5.

При данной конфигурации минимальное время convergence составило 0.4 с.

Полученные результаты демонстрируют, что при проектировании распределенных систем необходимо учитывать компромисс между скоростью распространения информации и сетевой нагрузкой, а также влияние географических задержек на работу Gossip-протоколов.
