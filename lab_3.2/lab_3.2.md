# 🔬 Методология

В работе использовалась симуляция Gossip-протокола для распределенной системы, состоящей из нескольких географических кластеров.

Система включает:
- 200 узлов;
- 4 кластера по 50 узлов;
- обмен gossip-сообщениями между узлами;
- случайные потери пакетов;
- моделирование отказов узлов;
- межкластерные задержки передачи сообщений.

Исследовалось влияние следующих параметров:
- Gossip Interval — частота обмена сообщениями;
- Gossip Fanout — количество соседей для gossip-рассылки;
- Packet Loss — процент потерянных сообщений;
- межкластерная задержка.

Для моделирования использовался класс `GeoSerfSimulator`, расширяющий базовую модель Gossip-протокола с учетом межкластерных задержек.

## Анализ графика

На графике показано влияние Gossip Interval и Fanout на время полной конвергенции системы.

### Наблюдения

- При увеличении Gossip Interval время convergence увеличивается.
- Fanout = 5 обеспечивает более быстрое распространение информации.
- Минимальное время convergence достигается при Interval = 0.2 с.

### Причина

При меньшем Gossip Interval узлы чаще обмениваются сообщениями, поэтому информация об отказе распространяется быстрее.

Больший Fanout позволяет каждому узлу отправлять gossip большему числу соседей, что также ускоряет распространение информации.

На графике показано влияние параметров Gossip-протокола на сетевую нагрузку.

- При уменьшении Gossip Interval количество сообщений увеличивается.

- Максимальный трафик наблюдается при маленьком интервале и большом fanout.

Это ускоряет convergence, но повышает нагрузку на сеть.
