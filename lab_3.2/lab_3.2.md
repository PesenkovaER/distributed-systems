| Параметр | Значение |
|----------|----------|
| Gossip Interval | 0.2, 0.5, 1.0 сек |

| Задержка между кластерами | 200 мс |

**Задача:** Исследовать влияние географической распределенности (задержки между кластерами) на время обнаружения отказов и нагрузку на сеть.

---

### 🔬 Методология

В работе используется дискретное событийное моделирование (Discrete Event Simulation) протокола Gossip. Симуляция включает:

1. **4 географических кластера** по 50 узлов
2. Задержка доставки сообщений:
   - Внутри кластера: 0 мс
   - Между кластерами: 200 мс
3. Вероятностная потеря пакетов (5% и 10%)
4. Случайный выбор соседей для обмена (fanout = 3 или 5)
Для корректного моделирования был разработан специальный класс `GeoSerfSimulator`, который учитывает принадлежность узлов к кластерам и соответствующие задержки.

### 💡 Ключевые выводы по географическим кластерам

1. **Задержка 200 мс является доминирующим фактором** — увеличивает время конвергенции в 3-15 раз

2. **"Волновой эффект"** — информация быстро распространяется внутри кластера (0 мс), но медленно между кластерами (200 мс за переход)

3. **Оптимальная конфигурация** для географически распределенных систем:
   - `Interval = 0.2с`, `Fanout = 5`
   - Время конвергенции: ~3.2с
   - Трафик: ~3980 сообщений

4. **Компромиссный вариант**:
   - `Interval = 0.2с`, `Fanout = 3`
   - Время конвергенции: ~4.8с (на 50% медленнее)
   - Трафик: ~2450 сообщений (на 38% меньше)

5. **Рекомендации**:
   - Для географически распределенных систем использовать более агрессивный fanout (4-6)

### 📝 Общие выводы по работе

1. Протокол Gossip эффективно справляется с обнаружением отказов в распределенных системах, обеспечивая надежность за счет избыточности.

2. **Ключевые параметры** влияющие на производительность:
   - **Gossip Interval** — обратно пропорционален скорости и прямо пропорционален трафику
   - **Gossip Fanout** — увеличение ускоряет конвергенцию, но сильно повышает нагрузку на сеть
   - **Packet Loss** — до 10% потерь система сохраняет устойчивость

3. **Географическая распределенность** (задержка 200 мс между кластерами) увеличивает время конвергенции в 3-15 раз.

4. **Для варианта 15** оптимальным балансом скорости и трафика является конфигурация: `Interval = 0.2 с`, `Fanout = 5`.

5. **Для максимальной скорости** (приоритет обнаружения сбоя) следует использовать: `Interval = 0.1 с`, `Fanout = 10`.

---

   - Интервал не должен превышать 0.3с для минимизации влияния задержки
   - При возможности использовать иерархический (многоуровневый) gossip
